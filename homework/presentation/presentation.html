Nested and Description Lists

Nested Lists

Just about anything fits in a list, from sets of instructions to collections of links. You can create subcategories by nesting lists within lists So basically nested lists are list inside lists.

This is an example of what the nested list looks like in code

The Basics

  • All nested lists follow the same markup pattern
    • Each list item in the top-level ordered list is followed by a complete second-level list
    • The second level lists sit inside the top-level list, not in the list items.
  • Nested Lists use the basic list tags

This is what a Nested list would look like in the broswer

Most Common Uses

  • Site Maps and other navigation tools
  • Tables of content for online books and papers
  • Outlines
  • To do lists

Description Lists

A Description list is a list of terms, with a description of each term.

Description lists don't use the basic list tags,there are specific ones that you use when you are making a descripiton list

  • Tags
    • dl tag defines the description list
    • dt tag defines the term(name)
    • dd tag describes each term
